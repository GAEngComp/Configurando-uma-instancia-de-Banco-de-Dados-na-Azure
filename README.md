# Configurando-uma-instancia-de-Banco-de-Dados-na-Azure
Desafio de Projeto DIO - Configurando uma instância de Banco de Dados na Azure

**Introdução**

Dentro dos ambientes de nuvem, como o da Microsoft Azure, existem 3 tipos de serviço, os quais possuem tanto pontos positivos quanto pontos negativos, além de que há situações em que um é melhor que o outro. São eles: Infraestrutura como Serviço (IaaS), Plataforma como Serviço (PaaS) e Software como Serviço (SaaS).

**Descrição dos serviços**

Abaixo serão descritos cada um dos tipos de serviço de nuvem:

**- Infraestrutura como Serviço:** se refere à criação de uma infraestrutura de TI a partir do pagamento por demanda de máquinas virtuais, servidores, redes, armazenamento e sistemas operacionais de um determinado provedor de nuvem. Embora seja um serviço bastante fléxivel em que o usuário pode gerenciar e configurar o hardware como quiser, os custos e trabalhos envolvidos são bastante altos, visto que é necessário, justamente, a configuração de cada parte do hardware e, após isso, manuntenção e vigia sobre o mesmo para garantir sua funcionalidade.

**- Plataforma como Serviço:** se refere à criação e aplicação de softwares na nuvem sem necessidade de gerenciamento de infraestrutura, já que está é gerenciada pelo provedor. É um tipo de serviço bastante utilizado no desenvolvimento de aplicativos, visto que o usuário precisa manusear apenas o software que desenvolverá, enquanto o provedor gerencia o hardware.

**- Software como Serviço:** se refere ao uso de aplicativos que estão na nuvem a partir da internet. É o tipo de serviço que menos demanda do usuário, pois se baseia em um modelo de assinatura, ou seja, o usuário paga conforme o uso e não precisa se preocupar nem com o gerenciamento do hardware, nem com o do software.


**Conclusão**

Em geral, todos os tipo de serviço de nuvem são úteis a sua própria maneira, ou seja, dependendo do contexto, um tipo de serviço se sobrepõe ao outro. Todavia, é importante ressaltar que é necessário uma cuidadosa avaliação prévia de qual deles se encaixam melhor na situação que o usuário necessita para evitar gastos desnecessários e/ou um serviço incompleto ou ineficiente para as tarefas designadas.
